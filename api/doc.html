Getting your token

If it doesn't show a token, that means you don't have one. Click the reset button to make one.

Fetching statuses

GET https://status.ryslig.xyz/api/timeline

Parameters:

type
Possible options: public, profile, mentions, following, specific. Public is default.
user
Only needed when type is profile, mentions, or following.
id
Only needed when type is specific. Used for permalinks.
page
Self-explanatory. Default is 1.
limit
Defines the amount of updates. Cannot exceed 100. Default is 25.

Example: https://status.ryslig.xyz/api/timeline?type=profile&user=status&limit=5

Getting information about a user

GET https://status.ryslig.xyz/api/profile

user*
The username of the person you want information on.
token
The token of the person you want information on. Can be used instead of the user parameter.

Example: https://status.ryslig.xyz/api/profile?user=status

Updating your status

POST https://status.ryslig.xyz/api/update

Note: You can only update your status once every 30 seconds.

token*
Self-explanatory.
status*
A string. Must be longer than two characters and shorter than 200 characters.
reply
The ID of the status you want to reply to.

Deleting your status

GET https://status.ryslig.xyz/api/delete

token*
Self-explanatory.
id*
The ID of the status you want to delete.

Example: https://status.ryslig.xyz/api/delete?token=d9ad4029e23ad319&id=5

This probably doesn't make that much sense. If you have any questions, feel free to reach out.

Also, the red asterisk means that the parameter is required.
